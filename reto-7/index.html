TypeScript

Typescript es un lenguaje de programación que extiende JavaScript agregando tipos estáticos opcionales. Esto significa que podemos definir tipos para las variables, funciones y objetos, lo que proporciona una mayor robustez y seguridad en nuestro código.

Tipos de datos

  • Any: Es un tipo que puede representar cualquier valor de JavaScript sin restricciones.
    let dynamicValue: any = "Hello";
    dynamicValue = 42; // No hay error en tiempo de compilación

    • Datos primitivos

    • String: Representa una cadena de texto y se declara usando comillas simples o dobles.
      let firstName: string = "John"
    • Number: Representa un número y puede ser de tipo entero o decimal.
      let age: number = 30;
      let pi: number = 3.14;
    • Boolean: Representa un valor verdadero o falso (true o false).
      let isStudent: boolean = true;
    • Void: Indica que una función no devuelve ningún valor.
      function logMessage(message: string): void { console.log(message); }
    • Null y Undefined: Representan valores nulos y no definidos respectivamente.
      let nullValue: null = null;
      let undefinedValue: undefined = undefined;

    Enumeraciones

    • Enum: Ofrecen una manera sencilla de trabajar con conjuntos de constantes relacionadas. Usos:
      • Permite reducir los errores que provoca la transposición o la escritura incorrecta de números.
      • Facilita el cambio de valores en el futuro.
      • Facilita la lectura del código, lo que significa que es menos probable que se produzcan errores en él.
      • Garantiza la compatibilidad con versiones posteriores. Con las enumeraciones, es menos probable que se produzca un error en el código si en el futuro alguien cambia los valores correspondientes a los nombres de miembro.
      enum Color { Red, Green, Blue, }
      let selectedColor: Color = Color.Green;

    Unknown y su diferencia con Any

    • Unknown: Similar a any, es similar al tipo any en que cualquier valor se puede asignar al tipo unknown. Sin embargo, no se puede acceder a las propiedades de un tipo unknown; tampoco se pueden llamar ni construir.
      let uncertainValue: unknown = "Hello";
      if (typeof uncertainValue === "string") { let length: number = uncertainValue.length; // Comprobación de tipo requerida }

      • La diferencia principal entre any y unknown es que no puede interactuar con una variable de tipo unknown; si lo hace, se genera un error del compilador. any omite las comprobaciones en tiempo de compilación y el objeto se evalúa en tiempo de ejecución. Si el método o la propiedad existen, se comportará según lo esperado.

    Unión e Intersección

    • Unión (|): Permite declarar una variable con múltiples tipos posibles.
      let value: string | number = "hello";
      value = 42;
    • Intersección (&): Permite combinar varios tipos en uno.
      type Person = { name: string; };
      type Employee = { empId: number; };
      type EmployeePerson = Person & Employee;

    Tipos de Colección

    • Array: Representa una colección ordenada de elementos del mismo tipo.
      let numbers: number[] = [1, 2, 3, 4, 5];
      let fruits: string[] = ["apple", "banana", "orange"];
    • Tuple: Representa un array con un número fijo de elementos, cada uno con un tipo específico.
      let coordinates: [number, number] = [10, 20];

    Buenas prácticas

    • Utilizar tipos explícitos para variables y funciones para mejorar la legibilidad y mantenibilidad del código.
    • Evitar el uso excesivo de any, preferir usar tipos más específicos siempre que sea posible.
    • Utilizar uniones y tipos discriminantes para representar variantes de datos.
    • Emplear enumeraciones en lugar de valores literales para evitar errores tipográficos.
    • Utilizar tipos genéricos para escribir funciones y clases reutilizables.
